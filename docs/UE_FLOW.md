# UE 交互逻辑

## 1. 登录流程

```
[开始] --> [登录页 /login]
                |
                v
          [输入用户名+密码]
                |
                v
          [点击登录]
                |
                v
          [前端校验]
                |
        +-------+-------+
        |               |
        v               v
    [校验通过]      [校验失败]
        |               |
        v               v
  [调用登录API]   [显示错误提示]
        |               |
  +-----+-----+        |
  |           |        |
  v           v        |
[成功]     [失败]      |
  |           |        |
  v           v        |
[存token   [显示错误  |
 到localStorage 提示] |
 跳转 /dashboard]     |
  |                    |
  v                    |
[结束] <--------------+
```

### 校验规则
- 用户名：非空
- 密码：非空

### 错误处理
- 401：显示"用户名或密码错误"
- 网络错误：显示"网络异常，请稍后重试"

## 2. 路由守卫逻辑

```
[访问任意路由]
        |
        v
  [localStorage中有token?]
        |
  +-----+-----+
  |           |
  v           v
 [有]       [无]
  |           |
  v           v
[目标页是   [跳转 /login]
 /login?]
  |
  +-----+-----+
  |           |
  v           v
 [是]       [否]
  |           |
  v           v
[跳转       [放行，加载目标页]
 /dashboard]
```

### 权限判断
- admin 角色：可访问所有路由
- tech 角色：只能访问 /dashboard、/orders、/orders/:id
  - /technicians、/customers、/orders/create 禁止访问，跳转 /dashboard

---

## 3. 创建工单流程

```
[开始] --> [点击"新建工单"按钮]
                |
                v
          [进入 /orders/create]
                |
                v
          [填写表单]
          - 客户姓名(必填)
          - 客户电话
          - 维修地址(必填)
          - 问题描述(必填)
          - 指派技师(下拉选择，可空)
                |
                v
          [点击提交]
                |
                v
          [前端校验]
                |
        +-------+-------+
        |               |
        v               v
    [校验通过]      [校验失败]
        |               |
        v               v
  [调用创建API]   [标记错误字段]
        |               |
  +-----+-----+        |
  |           |        |
  v           v        |
[成功]     [失败]      |
  |           |        |
  v           v        |
[提示"创建 [显示错误  |
 成功"]     提示]     |
  |                    |
  v                    |
[跳转 /orders] <------+
```

### 表单交互
- 指派技师下拉：从 GET /api/technicians 加载，显示"姓名 (在单N个)"
- 可选择"暂不指派"选项
- 选择技师后，工单创建后状态自动为 working；未选择则为 pending

---

## 4. 工单状态流转交互

```
[工单详情页 /orders/:id]
        |
        v
  [当前状态: pending]
        |
        v
  [显示"指派技师"按钮]
        |
        v
  [弹出技师选择弹窗]
        |
        v
  [选择技师 + 确认]
        |
        v
  [调用 PUT /api/orders/:id/status {status:"working", tech_id:x}]
        |
        v
  [状态变为 working]
        |
        v
  [显示"标记完工"按钮 + "重新派单"按钮]
        |
  +-----+-----+
  |           |
  v           v
[标记完工] [重新派单]
  |           |
  v           v
[录入费用] [调用PUT status:"pending"]
[调用PUT       |
 status:"done" v
  |       [状态变回 pending]
  v
[状态变为 done]
  |
  v
[显示"确认结算"按钮]
  |
  v
[确认费用 + 调用 PUT status:"settled"]
  |
  v
[状态变为 settled]
  |
  v
[无更多操作按钮]
```

### 按钮显示规则
| 当前状态 | 可用操作 | 按钮样式 |
|----------|---------|---------|
| pending | 指派技师 | primary |
| working | 标记完工 + 重新派单 | success + warning |
| done | 确认结算 | primary |
| settled | 无 | 灰色文字"已结算" |

## 5. 工单列表交互

```
[进入 /orders]
        |
        v
  [加载工单列表 GET /api/orders]
        |
        v
  [显示表格 + 筛选区]
        |
        v
  [操作选择]
  |
  +--- [切换状态Tab] --> [重新请求列表 + 更新表格]
  |
  +--- [输入搜索词] --> [防抖500ms + 重新请求]
  |
  +--- [点击表格行] --> [跳转 /orders/:id]
  |
  +--- [翻页] --> [重新请求 page 参数]
  |
  +--- [点击导出] --> [调用 GET /api/orders/export --> 下载文件]
```

### 筛选Tab
- 全部 | 待派单 | 进行中 | 已完工 | 已结算
- Tab 切换传递 status 参数

### 搜索
- 防抖 500ms
- 搜索客户名、地址、问题描述

---

## 6. 数据看板交互

```
[进入 /dashboard]
        |
        v
  [加载统计数据 GET /api/dashboard/stats]
        |
        v
  [渲染4个统计卡片]
  - 待派单(红色) | 进行中(橙色) | 已完工(绿色) | 已结算(灰色)
        |
        v
  [渲染折线图]
  - X轴：近7天日期
  - Y轴：工单数量
  - 面积填充效果
        |
        v
  [渲染饼图]
  - 4个状态对应的颜色
  - hover 显示具体数量和百分比
        |
        v
  [点击统计卡片]
  --> [跳转 /orders?status=对应状态]
```

## 7. 技师管理交互

```
[进入 /technicians]
        |
        v
  [加载技师列表 GET /api/technicians]
        |
        v
  [显示技师表格]
  - 姓名 | 手机 | 在单数 | 操作
        |
        v
  [操作选择]
  |
  +--- [新增技师] --> [弹出表单Dialog] --> [提交 POST /api/technicians] --> [刷新列表]
  |
  +--- [编辑] --> [弹出表单Dialog(预填)] --> [提交 PUT /api/technicians/:id] --> [刷新列表]
  |
  +--- [删除] --> [确认弹窗] --> [调用 DELETE] --> [刷新列表]
                                      |
                              +-------+-------+
                              |               |
                              v               v
                          [成功]          [有在单工单]
                              |               |
                              v               v
                          [删除成功]     [提示"该技师有
                          [刷新列表]     进行中工单，无法删除"]
```

## 8. 客户管理交互

```
[进入 /customers]
        |
        v
  [加载客户列表 GET /api/customers]
        |
        v
  [显示客户表格 + 搜索框]
        |
        v
  [操作选择]
  |
  +--- [新增客户] --> [弹出表单Dialog] --> [提交 POST /api/customers] --> [刷新列表]
  |
  +--- [编辑] --> [弹出表单Dialog(预填)] --> [提交 PUT /api/customers/:id] --> [刷新列表]
  |
  +--- [删除] --> [确认弹窗] --> [调用 DELETE] --> [刷新列表]
  |
  +--- [搜索] --> [防抖500ms + 重新请求]
```

### 新建工单时快捷选择客户
- 创建工单页面，客户姓名输入框旁有"选择客户"按钮
- 点击弹出客户列表弹窗，选择后自动填入姓名、电话、地址
